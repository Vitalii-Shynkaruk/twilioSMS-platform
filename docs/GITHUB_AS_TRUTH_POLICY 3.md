# GitHub-as-Truth Policy

© BuyReadySite.com

## Правило

Production меняется только через GitHub flow: feature branch -> PR -> review -> merge -> deploy.

Прямые правки на прод-сервере запрещены.

## Обязательные требования

1. Каждый прод-фикс оформляется коммитом в репозитории.
2. В PR должны быть:
   - цель изменения;
   - список рисков;
   - шаги smoke проверки;
   - план отката.
3. Любые emergency hotfix на сервере:
   - разрешены только для остановки инцидента;
   - в течение 24 часов обязательно перенос в репозиторий отдельным PR.

## Контроль соблюдения

1. Перед релизом проверяется, что рабочее дерево чистое или содержит только ожидаемые изменения.
2. Деплой выполняется только из зафиксированного commit SHA.
3. После релиза фиксируется SHA и время в release notes.

## Нарушения

Любой обнаруженный direct prod edit без PR считается нарушением release policy и требует post-mortem.
